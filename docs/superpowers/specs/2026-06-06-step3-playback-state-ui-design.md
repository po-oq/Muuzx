# Step 3: 再生状態付き一覧 UI Design

## 1. 目的

Step 3 では、Step 2 までに作ったフォルダ取り込みとローカル再生を土台に、ファイル一覧とミニプレイヤーを `docs/ui-mock.html` に可能な限り忠実に再現する。

見た目を整えるだけではなく、再生位置、総再生時間、未再生・途中・再生済み状態をメモリ上で実際の再生に連動させる。通常タップで途中位置から再開でき、長押しから「未再生に戻す」「先頭から再生」を実行できる状態までを Step 3 の完了範囲とする。

再生位置と状態の JSON 永続化は Step 4 で行う。Step 3 では画面を開いている間と、同一アプリ起動中に一覧を開き直した場合に状態を維持するが、アプリ終了・再起動後の復元は行わない。

## 2. MVP Roadmap

Step 3 以降で全体像を見失わないよう、MVP までの切り方をここに固定する。

1. Step 1: ローカル再生（完了）
2. Step 2: フォルダ画面 + フォルダ選択 + ローカル取り込み（完了）
3. Step 3: 一覧/ミニプレイヤーを `ui-mock.html` に忠実に再現 + メモリ上の再生状態管理
4. Step 4: 再生位置/状態の JSON 永続化 + 起動時復元 + 保険保存
5. Step 5: Now Playing 画面 + バックグラウンド再生 + ロック画面/コントロールセンター操作
6. Step 6: iCloud Documents による再生状態同期
7. Step 7: フォルダ再指定/ワンタップ再同期/フォルダ画面とエラー処理の仕上げ

各 Step は画面上で動作確認できる単位にする。後続 Step の機能を先取りして Step 3 を大きくしない。

## 3. Step 3 の範囲

Step 3 で作るもの。

- `AudioListView` を `ui-mock.html` のファイル一覧に近づける。
- `MiniPlayerView` を `ui-mock.html` の下部プレイヤーに近づける。
- 全ファイルの総再生時間をバックグラウンドで順次取得する。
- 再生中の位置を約 1 秒間隔で画面へ反映する。
- 再生位置に応じて `unplayed`、`inProgress`、`played` を更新する。
- 通常タップで記録済み位置から再生を開始する。
- 長押しメニューから「未再生に戻す」「先頭から再生」を実行する。
- 再生完了後に現在項目を再生済みにして、次項目を自動再生する。
- 一覧タイトルへ前回取り込み元フォルダ名を表示する。

## 4. Step 3 では作らないもの

以下は意図的に後続 Step へ送る。

- 再生位置、総再生時間、未再生・途中・再生済み状態の JSON 保存と起動時復元（Step 4）
- 再生中の 30〜60 秒ごとのローカル保険保存（Step 4）
- Now Playing 詳細画面、再生速度変更 UI（Step 5）
- バックグラウンド再生、Now Playing Info、ロック画面/コントロールセンター操作（Step 5）
- iCloud Documents による端末間の再生状態同期（Step 6）
- フォルダ画面の `ui-mock.html` 忠実再現（Step 7）
- 保存済み bookmark を使うワンタップ再同期、フォルダ再指定モード選択（Step 7）
- 容量不足、stale bookmark、同期失敗などのエラー UI 仕上げ（Step 7）
- 進捗バーをドラッグする任意位置シーク
- Now Playing 画面への遷移

## 5. 設計方針

操作基盤には SwiftUI と iOS 標準コントロールを使い、見た目は `ui-mock.html` に可能な限り忠実に寄せる。

- ナビゲーション、ボタン、コンテキストメニュー、SF Symbols、Dynamic Type、アクセシビリティは iOS 標準を使う。
- 標準 `List` の外観には依存せず、`ScrollView` と `LazyVStack` で角丸の白いファイル一覧を構成する。
- 状態バッジ、細い進捗バー、背景色、余白、文字階層を明示的にスタイリングする。
- ミニプレイヤーは一覧画面の下部へ固定し、動的内容で高さが変化しないよう安定した寸法を持たせる。
- Step 4 の永続化を接続しやすいよう、状態変更の責務は `AudioListViewModel` に集約する。

## 6. コンポーネント設計

### AudioMetadataService

責務。

- ローカル音声ファイル URL を受け取る。
- `AVURLAsset` から総再生時間を非同期取得する。
- 取得できた秒数を返す。
- 読み取れないファイルは失敗として返し、呼び出し側が時間表示を省略できるようにする。

一覧の初期表示はメタデータ取得を待たない。ファイル項目をすぐ表示し、総再生時間はバックグラウンドで順次反映する。

### PlaybackService

既存の再生責務に、Step 3 で必要な最小限の操作を追加する。

- 指定項目をその項目の `positionSec` から再生する。
- 指定項目を 0 秒から再生する。
- 現在位置と総再生時間を公開する。
- 現在項目を停止し、選択状態を解除する。
- 10 秒戻し・30 秒送り後の現在位置を取得できるようにする。

再生状態モデルの更新や UI 用タイマーは `PlaybackService` に持たせない。

### AudioListViewModel

Step 3 の再生セッション状態を管理する中心とする。

- `AudioMetadataService` から取得した総再生時間を各 `AudioItem` へ反映する。
- 再生開始、一時停止、スキップ、完了、長押し操作に応じて項目を更新する。
- 再生中は約 1 秒間隔で現在位置を取得し、一覧とミニプレイヤーへ反映する。
- 再生位置から `PlaybackStatus` を判定する。
- 現在再生中の項目 ID と再生中フラグを管理する。
- 一覧再読み込み時に、同一アプリ起動中の既存状態を同じ `fileId` の項目へ引き継ぐ。
- 画面離脱や一覧再読み込み時に、古いメタデータ取得タスクと位置更新タスクを停止する。

Step 4 では、この ViewModel が発生させる状態変更イベントを JSON 保存へ接続する。

### AudioListView

- 背景は淡いグレーにする。
- ナビゲーションタイトルは取り込み元フォルダ名とする。取得できない場合は `ファイル一覧` と表示する。
- `ファイル一覧` のセクションラベルを表示する。
- 角丸の白いコンテナ内へファイル行を自然順で並べる。
- 行タップで通常再生を開始する。
- 標準コンテキストメニューで「未再生に戻す」「先頭から再生」を表示する。
- 長いファイル名、Dynamic Type、状態バッジが重なりにくいレイアウトにする。

### AudioFileRow

一覧行は独立した小さな View とし、次を表示する。

- ファイル名
- 状態と時間の補助テキスト
- 表示専用の細い進捗バー
- 状態バッジ

表示ルール。

- 未再生: ファイル名を太字、補助テキストは `未再生` または `未再生・総時間`、バッジは `未再生`
- 途中: 補助テキストは `途中・現在位置 / 総時間`、バッジは進捗率
- 再生済み: 補助テキストは `完了・総時間`、バッジは `100%`
- 現在再生中: 状態バッジより優先して `再生中` バッジを表示
- 総時間不明: 時間部分と進捗率を省略する

### MiniPlayerView

- 画面下部へ固定する。
- 現在ファイル名、表示専用進捗バー、現在位置、総時間を表示する。
- 10 秒戻し、再生/一時停止、30 秒送りを表示する。
- 未選択時も高さを維持し、`再生していません` を表示する。
- 未選択時は再生操作を無効化する。
- Step 3 では進捗バーのドラッグ操作を実装しない。

## 7. 状態遷移

状態判定は `docs/sync-spec.md` と一致させる。

### unplayed

- `positionSec` は 0。
- 通常タップまたは先頭から再生で再生を開始すると `inProgress` へ移る。

### inProgress

- 再生位置が 0 より大きく、再生済み判定を満たさない。
- 一時停止、10 秒戻し、30 秒送り、約 1 秒間隔の更新で位置を反映する。

### played

以下のいずれかで再生済みとする。

- 再生完了イベントを受け取った。
- 総再生時間が既知で、`positionSec >= durationSec - 30` を満たした。

再生済みへ移るときは、表示上の `positionSec` を `durationSec` へ揃える。総時間不明の場合、残り 30 秒判定はせず、再生完了イベントでのみ再生済みにする。

### 長押し操作

#### 未再生に戻す

- 対象項目を `positionSec = 0`、`status = unplayed` にする。
- 対象が現在再生中なら再生を停止し、現在項目の選択状態も解除する。

#### 先頭から再生

- 対象項目を `positionSec = 0` にする。
- 対象項目を直ちに再生する。
- 状態は再生開始後に `inProgress` とする。

### 通常タップ

- 項目の `positionSec` から再生を開始する。
- 途中状態なら途中位置から再開する。
- 再生済み項目を通常タップした場合も、記録された末尾位置からではなく 0 秒から再生する。

## 8. データフロー

### 一覧表示

1. `AudioListViewModel.load()` がローカル音声一覧を読み込む。
2. 同一アプリ起動中に保持している状態を `fileId` で新しい一覧へ引き継ぐ。
3. 一覧をすぐ画面へ表示する。
4. `AudioMetadataService` が各ファイルの総再生時間をバックグラウンドで順次取得する。
5. 取得できた項目だけ `durationSec` と表示を更新する。

### 再生

1. ユーザーが項目を通常タップする。
2. `AudioListViewModel` が開始位置を決め、`PlaybackService` へ再生を依頼する。
3. `AudioListViewModel` が位置更新タスクを開始する。
4. 約 1 秒ごとに位置と状態を更新する。
5. 一覧行とミニプレイヤーが同じ状態を表示する。

### 再生完了

1. `PlaybackService` が再生完了を通知する。
2. `AudioListViewModel` が完了した項目を `played` に更新する。
3. 次項目があれば 0 秒から自動再生する。
4. 次項目を現在再生中として表示する。

## 9. 非同期処理とライフサイクル

- 総再生時間の取得はメインスレッドをブロックしない。
- 一覧を再読み込みした場合、古いメタデータ取得結果が新しい一覧を上書きしないようタスクをキャンセルする。
- 再生位置更新タスクは再生中だけ動作する。
- 一時停止時は最後の位置を即時反映してから位置更新タスクを停止する。
- 再開時は位置更新タスクを再開する。
- 現在項目が一覧から消えた場合は、再生を停止して選択状態を解除する。

## 10. エラー処理

Step 3 のエラー処理はローカル再生を妨げない範囲に留める。

- 総再生時間の取得失敗: ユーザーへエラー表示せず、その項目の時間と進捗率を省略する。
- 総再生時間が 0 または不明: 残り 30 秒判定を行わない。
- 記録位置が総再生時間を超える: 再生開始位置を総再生時間以内へ丸める。
- 現在項目が一覧再読み込みで消えた: 再生を停止して現在項目を解除する。
- 音声再生自体の詳細なエラー表示は後続 Step の仕上げへ送る。

## 11. テスト方針

### AudioMetadataServiceTests

- 音声ファイルから総再生時間を取得できる。
- 読み取れないファイルで失敗を返す。

AVFoundation の実ファイル依存を抑えるため、必要に応じてメタデータ取得プロトコルを通して ViewModel テストでは fake を使う。

### PlaybackServiceTests

- 項目の途中位置から再生を開始する。
- 項目を 0 秒から再生する。
- 記録位置を総再生時間以内へ丸める。
- 現在項目を停止して選択解除する。
- 再生完了後に次項目へ進む。

### AudioListViewModelTests

- 一覧を待たずに表示し、総時間を後から反映する。
- 同一 `fileId` のメモリ内状態を再読み込み後も引き継ぐ。
- 再生位置更新で `unplayed` から `inProgress` へ変わる。
- 残り 30 秒以内で `played` へ変わる。
- 総時間不明では残り 30 秒判定を行わない。
- 再生完了で対象を `played` にし、次項目を再生する。
- 通常タップで途中位置から再生する。
- 「先頭から再生」で 0 秒から再生する。
- 「未再生に戻す」で位置と状態を初期化する。
- 現在再生中の項目を未再生へ戻すと停止・選択解除する。
- 一時停止、スキップ操作後に表示位置を更新する。

### UI Tests

- 一覧に未再生表示とミニプレイヤーが表示される。
- 項目タップ後に `再生中` バッジとミニプレイヤー内容が更新される。
- 長押しメニューに「未再生に戻す」「先頭から再生」が表示される。
- ミニプレイヤーの再生/一時停止、10 秒戻し、30 秒送りを操作できる。

### 手動スモーク

- `docs/ui-mock.html` と見比べ、一覧、状態バッジ、進捗バー、ミニプレイヤーの見た目を確認する。
- 長いファイル名と Dynamic Type で表示が破綻しないことを確認する。
- 再生中に位置・状態・進捗が変化することを確認する。
- 途中位置からの再開、先頭から再生、未再生化を確認する。

## 12. 完了条件

Step 3 は以下を満たしたら完了とする。

- ファイル一覧とミニプレイヤーが `ui-mock.html` に可能な限り忠実な見た目になっている。
- 一覧は総再生時間の取得を待たずに表示される。
- 総再生時間がバックグラウンドで順次反映される。
- 再生中の現在位置、進捗、状態が画面上で更新される。
- 未再生、途中、再生済み、再生中を一覧で判別できる。
- 通常タップで途中位置から再開できる。
- 長押しから「未再生に戻す」「先頭から再生」を実行できる。
- 再生完了または残り 30 秒以内で再生済みになる。
- 再生完了後に次項目へ自動再生できる。
- 進捗バーは表示専用である。
- フォルダ画面、Now Playing 画面、永続化、バックグラウンド再生、iCloud 同期へ不用意に踏み込んでいない。
- unit/UI tests が通る。
- シミュレータで手動スモークが通る。
