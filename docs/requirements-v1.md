# Audio Folder Player 要件定義 v1

## 1. 目的

iCloud Drive上の任意フォルダに置いた音声ファイルを、iPhone/iPadへ取り込み、ローカル再生する。

GoodReaderのようなファイル一覧ベースの操作感を維持しつつ、再生位置・未再生状態を複数端末で同期できるようにする。

## 2. 想定ユーザー

- iPhoneでウォーキング中に音声ファイルを聴くユーザー
- 自宅ではiPadとBluetoothスピーカーで音声ファイルを聴くユーザー
- mp3/m4aなどの音声ファイルをフォルダ単位で管理しているユーザー
- Apple Music/Podcast的なライブラリ管理ではなく、GoodReader風のファイル一覧UIを好むユーザー

## 3. 基本方針

```text
iCloud Drive = 音声ファイルの取り込み元
端末ローカル = 再生用キャッシュ（再生状態のsource of truth）
iCloud Documents = 再生状態の同期先（iCloud未設定時はローカルのみ）
```

音声ファイルは再生時にiCloudから直接ストリーミングしない。

指定されたiCloud Driveフォルダから端末ローカルのアプリ専用領域へコピーし、再生はローカルファイルを対象に行う。

## 4. 必須機能

### 4.1 iCloudフォルダ取り込み

- iCloud Drive上の任意フォルダを指定できる
- ファイル単体ではなく、フォルダを指定する
- 指定フォルダ内の音声ファイルをスキャンする
- 対応音声ファイルが0件の場合は取り込みしない
- iCloud上で未ダウンロード（プレースホルダ）のファイルは、コピー前にダウンロードを確保する
- 音声ファイルを端末ローカルのアプリ専用領域へ自動コピーする
- 取り込み中は進捗を表示する
- 取り込み元フォルダ名と最終取り込み日時を表示する

### 4.2 ローカル再生

- 再生はローカル保存済みファイルを対象にする
- ファイル名昇順で一覧表示する
- ファイル名昇順で連続再生する
- 1ファイルの再生完了後、自動で次ファイルを再生する
- 10秒戻しに対応する
- 30秒送りに対応する
- 再生速度変更はMVPでは任意だが、UI上は拡張可能にする

### 4.3 再生状態管理

各ファイルに以下の状態を持つ。

- 未再生
- 途中
- 再生済み

表示仕様。

- 未再生ファイルは太字表示する
- 途中ファイルは進捗バーを表示する
- 再生済みファイルは100%表示する
- 現在再生中のファイルには再生中表示を付ける
- 任意のファイルを未再生に戻せる
- 未再生に戻した場合、再生位置は0秒に戻す
- 任意のファイルを先頭から再生できる

### 4.4 複数端末同期

iPhone/iPad間で以下を同期する。

- 現在再生中のファイル
- 各ファイルの再生位置
- 各ファイルの未再生/途中/再生済み状態
- 最終更新日時

同期タイミング。

- 一時停止
- 曲送り
- 曲戻し
- 再生完了
- 未再生に戻す操作
- アプリがバックグラウンドへ移行するとき

競合解決はシンプルに「最新更新勝ち」とする。

同期は iCloud Documents（再生状態JSONを1ファイルで同期）で行う。iCloud未設定・オフライン時はローカル状態のみで動作し、同期は静かに無効化する（端末内で完結）。

詳細は `sync-spec.md` を参照する。

### 4.5 バックグラウンド再生

- 画面OFF中も再生を継続する
- 自動スリープ中も再生を継続する
- ロック画面から再生・停止・スキップ操作できる
- コントロールセンターから再生・停止・スキップ操作できる

### 4.6 フォルダ再指定

iCloudフォルダを再指定できる。

再指定時に以下を選択できる。

1. ローカル全削除して再取り込み
2. 既存を残して上書き/差分取り込み

#### ローカル全削除して再取り込み

- アプリ内のローカル音声ファイルを削除する
- ローカルの再生状態も初期化する
- 指定フォルダから音声ファイルを再取り込みする

#### 既存を残して上書き/差分取り込み

- 同名ファイルがある場合は上書きする
- 新規ファイルは追加する
- 既存ローカルファイルは原則残す
- 削除同期はMVPでは行わない

## 5. 画面要件

UIの具体イメージは `ui-mock.html` を参照する。

### 5.1 フォルダ画面

表示内容。

- 前回取り込みフォルダ名
- ファイル数
- 使用容量
- 最終同期日時
- 再同期ボタン（保存したブックマークから前回フォルダをワンタップ再取り込み）
- 別フォルダ選択ボタン（UIDocumentPickerを開く）
- 再生位置/未再生状態の同期状態（iCloud未設定時は「端末内のみ」と表示）

### 5.2 ファイル一覧画面

GoodReader風の一覧UIとする。

表示内容。

- ファイル名
- 未再生太字
- 進捗バー
- 再生中バッジ
- 再生時間/進捗
- 下部ミニプレイヤー

操作。

- タップで再生
- 長押しでメニュー表示
  - 未再生に戻す
  - 先頭から再生

### 5.3 下部ミニプレイヤー

ファイル一覧画面下部に常時表示する。

表示内容。

- 現在のファイル名
- 再生/一時停止
- 10秒戻し
- 30秒送り
- 現在位置 / 総時間
- 進捗バー

### 5.4 再生画面

必要に応じて表示する詳細再生画面。

表示内容。

- 現在のファイル名
- 取り込み元フォルダ名
- 再生/一時停止
- 10秒戻し
- 30秒送り
- 現在位置 / 総時間
- 進捗バー
- 再生速度ボタンの配置余地
- 未再生に戻す操作

## 6. 対応音声形式

MVP必須。

- mp3
- m4a

後続候補。

- aac
- wav

## 7. エラー要件

以下はユーザーに明示する。

- 指定フォルダに対応音声ファイルがない
- iCloud Driveにアクセスできない
- ファイルコピーに失敗した
- ローカル容量が不足している
- 再生状態の同期に失敗した
- 再生できないファイル形式である
- 取り込み中にアプリが中断された

## 8. 作らないもの

MVPでは以下は対象外。

- Podcast連携
- Apple Podcastsとの同期
- Apple Music連携
- プレイリスト作成
- ファイル名変更
- ファイル削除
- フォルダ作成
- タグ編集
- 音声ファイルのクラウドストリーミング再生
- 秒単位のリアルタイム同期
- Webアプリ対応
- iCloud Drive上の音声ファイルの編集
- iCloud Drive上のファイル削除

## 9. 推奨技術構成

```text
UI: SwiftUI
フォルダ選択: UIDocumentPicker
フォルダ権限永続化: security-scoped bookmark
ダウンロード確保: NSFileCoordinator
音声再生: 単一AVPlayer
バックグラウンド再生: AVAudioSession + Background Audio
ローカル保存: Application Support配下（audioはバックアップ除外）
状態保存: JSON
端末間同期: iCloud Documents（アプリのiCloudコンテナ内の状態JSONファイル）
```

詳細は `architecture.md` を参照する。

## 10. MVP完了条件

以下ができたらMVP完了とする。

- iCloud Driveの任意フォルダを選択できる
- 指定フォルダ内のmp3/m4aをローカルに取り込める
- 対応音声ファイルが0件の場合にエラー表示できる
- ファイル一覧から再生できる
- 曲終了後に次の曲へ進む
- 画面OFFでも再生継続する
- ロック画面/コントロールセンターから操作できる
- 未再生/途中/再生済みが表示できる
- 任意のファイルを未再生に戻せる
- iPhone/iPad間で再生位置と状態が同期される
- フォルダ再指定時に全削除/上書きを選べる

## 11. v1時点の設計判断

- iCloud上のアプリ専用フォルダは使わない
- 既存のiCloud Drive任意フォルダを取り込み元にする
- 再生は必ずローカルファイルから行う
- ファイル操作アプリではなく、音声再生アプリとして作る
- GoodReaderライクなファイル一覧UIを優先する
- Podcast対応はしない

### 技術レビュー反映（2026-06-04）

- 同期は `NSUbiquitousKeyValueStore` ではなく iCloud Documents を採用する（1MB上限回避）
- `SyncBackend` 抽象を入れ、将来CloudKitへ差し替え可能にする
- ローカル状態を source of truth とし、iCloud未設定・オフライン時はローカルのみで動作する
- 選択フォルダは security-scoped bookmark で永続化し、ワンタップ再同期を可能にする
- iCloud上の未ダウンロードファイルはコピー前にダウンロードを確保する（NSFileCoordinator）
- `fileId` は `正規化ファイル名 + サイズ`（durationは含めない）、ファイル名はNFC正規化必須
- `audio/` はバックアップ除外、取り込みは中断後に再開可能とする
